Pulling metadata from BIM models, and measuring distance & area on 3D Tiles.

CesiumJS 1.142   3D Tiles   Cesium ion

What we'll cover

  • 1 · Inspection — how we pull & show metadata from BIM features
  • 2 · Distance — point-to-point surface measurement
  • 3 · Area — polygon surface area in 3D

All three start from a single click on the model, dispatched by mode in js/features/dispatch.js.

Feature 1

Inspection

Click a BIM element → highlight it → read its per-feature metadata.

The core idea

  • A BIM tileset is made of Cesium3DTileFeature objects — one per element (wall, beam, pipe…).
  • Each feature carries property key/value pairs baked in during ion tiling (IFC/DGN/FBX attributes).
  • We never parse the source file — we read attributes straight off the picked feature at runtime.

js/features/inspect.js

Step 1 — Pick the feature

scene.pick() returns whatever is under the cursor. We only proceed if it's a BIM feature.


const picked = state.viewer.scene.pick(click.position);

if (!Cesium.defined(picked) ||
    !(picked instanceof Cesium.Cesium3DTileFeature)) {
  $("meta").classList.remove("show");   // clicked empty space / globe
  return;
}

Step 2 — Highlight it

Cache the original colour so we can restore it, then tint the feature.


state.highlighted = {
  feature: picked,
  color: Cesium.Color.clone(picked.color)   // remember original
};
picked.color = Cesium.Color.YELLOW.withAlpha(0.85);

// later, on close / mode change:
function clearHighlight() {
  if (state.highlighted)
    state.highlighted.feature.color = state.highlighted.color;
}

Step 3 — Pull the metadata

Two calls do all the work: list the keys, then read each value.


const ids = picked.getPropertyIds();          // e.g. ["element","category","Level"...]

let rows = "";
ids.sort().forEach((id) => {
  let val = picked.getProperty(id);           // the value for that key
  if (val && typeof val === "object") val = JSON.stringify(val);
  rows += `<tr><td>${id}</td><td>${val}</td></tr>`;
});
metaBody.innerHTML = `<table>${rows}</table>`;

getPropertyIds() + getProperty(id) → that's the whole metadata API.

Result

  • Side panel shows a clean key → value table.
  • Selected element glows yellow in the scene.
  • Works for any ion BIM asset that retained attributes.
  • Flow recap:
  • pick → feature
  • highlight → colour swap
  • getPropertyIds → keys
  • getProperty → values
  • render → table

⚠ The caveat that matters

  • ion's BIM tiling sometimes strips embedded attributes to slim the tiles.
  • If getPropertyIds() is empty, the geometry has no metadata.
  • Production pattern: keep an external metadata store keyed by element ID, and look it up using the feature's ID instead of relying on embedded properties.

The prototype shows a clear message when a feature carries no properties.

Feature 2

Distance

Click points on the surface → running poly-distance in metres.

Getting a 3D point from a click

  • scene.pickPosition() returns the exact Cartesian3 world point under the cursor — using the depth buffer, so it hits real geometry.
  • Requires pickPositionSupported (depth texture) — we guard for it.

const pos = scene.pickPosition(click.position);
if (!Cesium.defined(pos)) return;     // clicked empty space
state.measurePoints.push(pos);        // collect a Cartesian3[]

js/features/measure.js

Summing the segments

Straight-line 3D distance between each consecutive pair, accumulated.


let total = 0;
for (let i = 1; i < measurePoints.length; i++) {
  total += Cesium.Cartesian3.distance(
    measurePoints[i - 1],
    measurePoints[i]
  );
}
// total is in metres (ECEF is metric)
  • Each click extends the line → multi-segment paths supported.
  • A blue polyline + vertex dots visualise the path.
Feature 3

Area

Click 3+ points → closed polygon → surface area in m².

Area of a clicked 3D ring

  • Fan-triangulate from the first vertex — split the ring into triangles (v₀, vᵢ, vᵢ₊₁).
  • Each triangle's area = ½ · ‖a × b‖ (cross-product magnitude); sum them.
  • Exact for convex rings; an approximation for concave or strongly non-planar ones.

function polygonArea3D(pts) {
  let area = 0;
  const v0 = pts[0];
  for (let i = 1; i < pts.length - 1; i++) {
    const a = Cesium.Cartesian3.subtract(pts[i],     v0, new Cesium.Cartesian3());
    const b = Cesium.Cartesian3.subtract(pts[i + 1], v0, new Cesium.Cartesian3());
    const cross = Cesium.Cartesian3.cross(a, b, new Cesium.Cartesian3());
    area += Cesium.Cartesian3.magnitude(cross) * 0.5;
  }
  return area;   // m²
}

Drawing the polygon

  • The clicked ring is closed back to the first point (polyline).
  • A translucent fill uses perPositionHeight so it follows the real 3D shape — not draped on the ground.

viewer.entities.add({
  polygon: {
    hierarchy: measurePoints,
    perPositionHeight: true,              // honour each vertex's height
    material: Cesium.Color.fromCssColorString("#4cc2ff").withAlpha(0.18),
  }
});

One click, three tools

  • Inspect → scene.pick → feature properties
  • Distance → pickPosition → Σ segment lengths
  • Area → pickPosition → fan-triangle sum
  • Mode is held in state.mode and routed by dispatch.js.
  • Each feature lives in its own module under js/features/.
